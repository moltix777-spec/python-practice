📝 문제 6. 안전 나눗셈
- 핵심: 0으로 나눌 때 예외 처리, 정상 계산 시 반올림.
- 코드:contentReference[oaicite:1]{index=1}:
  if b == 0: return None
  return round(a/b, ndigits)
- 주의: f-string 포매팅(:.{ndigits}f)은 출력용일 때만. 연산용은 round() 결과를 리턴해야 수치로 활용 가능.

📝 문제 7. URL 생성 (**kwargs)
- 핵심: **kwargs로 받은 인수는 딕셔너리처럼 처리.
- 코드:contentReference[oaicite:2]{index=2}:
  query = "&".join(f"{k}={v}" for k, v in params.items())
  return f"{base}?{query}"
- 포인트: kwargs = {"q": "python", "page": 2} 형식. .items()로 key=value 조립.
- 주의: 이 문제는 kwargs 연습용, 실제 URL 인코딩은 따로 필요.

📝 문제 8. 피보나치 리스트
- 핵심: 리스트에 현재 항 a를 누적 + 동시에 a, b 갱신.
- 코드:contentReference[oaicite:3]{index=3}:
  seq = []
  a,b = 0,1
  for _ in range(n):
      seq.append(a)
      a,b = b, a+b
  return seq
- 포인트: for _ in range(n) → _ 는 “안 쓰는 변수” 관습.
- 주의: return 은 반드시 for문 바깥에서 실행.

📝 문제 9. 소수 판별 (√n 최적화)
- 핵심: 약수는 짝으로 존재 → √n까지만 확인해도 충분.
- 코드:contentReference[oaicite:4]{index=4}:
  if n < 2: return False
  for i in range(2, int(n**0.5)+1):
      if n % i == 0:
          return False
  return True
- 포인트: int(n**0.5) → 루트 n까지 확인.
- 주의: n < 2는 소수 아님 처리.

📝 문제 10. 주문 합계 계산
- 핵심: 문자열 "메뉴:개수,메뉴:개수" 파싱 후 메뉴판(dict)과 매칭 → 합계.
- 코드:contentReference[oaicite:5]{index=5}:
  menu = {"아메리카노":1500,"라떼":2000,"차":1200}
  total = 0
  orders = s.split(",")
  for order in orders:
      name, count = order.split(":")
      if name in menu:
          total += menu[name] * int(count)
  return total
- 포인트: split(",") → 주문 단위 나누기, split(":") → 이름/개수 분리.
- 주의: 메뉴판에 없는 건 무시, 공백 제거 로직 있으면 더 안전.

📌 핵심 정리 (6~10번)
- 안전 나눗셈: round()로 숫자 리턴, 0 나눗셈 예외 처리.
- URL 생성: kwargs → dict.items() 순회, 문자열 join으로 조립.
- 피보나치: 리스트 누적 + a,b 동시 갱신, return 위치 주의.
- 소수 판별: √n까지만 확인으로 최적화, n<2 예외 처리.
- 주문 합계: 문자열 파싱 후 dict 매칭, 없는 메뉴는 건너뛰기.
